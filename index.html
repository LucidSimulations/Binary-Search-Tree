TREE BASED DATA STRUCTURES

The following is definition of Binary Search Tree(BST) : Its is a node-based binary tree data structure which has the following properties:

  • The left subtree of a node contains only nodes with keys lesser than the node’s key.
  • The right subtree of a node contains only nodes with keys greater than the node’s key.
  • The left and right subtree each must also be a binary search tree.
  • There must be no duplicate nodes.

Searching a key
To search a given key in Bianry Search Tree, we first compare it with root, if the key is present at root, we return root. If key is greater than root’s key, we recur for right subtree of root node. Otherwise we recur for left subtree.

Insertion of a key
A new key is always inserted at leaf. We start searching a key from root till we hit a leaf node. Once a leaf node is found, the new node is added as a child of the leaf node.
Illustration to insert in the tree:
1. Start from root.
2. Compare the inserting element with root, if less than root, then recurse for left, else recurse for right.
3. After reaching end,just insert that node at left(if less than current) else right.

Deletion of a key
Following steps are followed in order to delete a key.
1. Traverse the tree to find the key. If key is found, do the following:-
 - If the key has neither left nor right child, simply delete it.
 - If the key has right child, find its inorder successor i.e. the smallest node in its right subtree, and replace the key with this successor after deleting it.
 - If the key has left child, find its inorder predecessor i.e. the largest node in its left subtree, and replace the key with this predecessor after deleting it.
2. Key is absent. So exit.

Simulation assumptions and constraints:-
  • Tree can be created upto level of 6.
  • Node can have integer or float values upto length of 6.
  • Next and previous steps are not implemented in delete operation.
